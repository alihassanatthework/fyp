1.   Introduction 
1.1 Purpose 
This document explains the goal of the AI-Based Skin and Hair Scalp Analysis System. The 
main purpose of this phase is to build a smart system that can look at a person’s skin and scalp 
image and find problems like acne, dandruff, or dryness using AI, It will also give personalized 
treatment according to their medical history if someone have diseases like (e.g diabetic, 
heart issues, pregnancy, etc.) so it cannot effect other diseases. This document helps 
developers, teachers, and project members understand what the system will do and how it will 
work. 
_____________________________________________________________________________________ 
 1.2   Scope 
In this first phase, the system focuses only on skin and scalp analysis. 
Users can upload or scan a photo of their face or hair, and the system will: 
• Find skin issues like acne, dark spots, or dryness. 
• Find scalp issues like dandruff, oiliness, or hair fall. 
• Show a simple report showing the detected problems, their severity level, and treatment 
recommendations based on the analysis. If the severity is high, then it suggests users 
consult with dermatologists. 
______________________________________________________________________________ 
 1.3   Definitions, Acronyms, and Abbreviations 
• AI (Artificial Intelligence): Smart computer technology that helps find skin or hair 
problems. 
• Scalp Analysis: Checking scalp health like dryness, oiliness, or dandruff. 
• Skin Condition: Any visible problem on the face like acne or dark spots. 
• Recommendation: Personalized treatment given by the system based on your results.

2   Overall Description  
2.1 Product Perspective 
It mainly focuses on checking the skin and scalp health of a person using AI. The user will upload or scan 
a picture of their face or hair, and the system will automatically find skin or scalp problems and suggest 
personalized treatment. 
____________________________________________________________________________________ 
2.2 Product Functions 
The main things this system can do are: 
• Take or upload a photo of your face or hair. 
• Find skin problems like acne, dark spots, or dryness etc. 
• Find scalp problems like dandruff, oiliness, or hair fall etc. 
• What problems were found and how serious they are and treatments such as (Acne: Use 
gentle face wash, aloe vera gel, Dark spots: Use vitamin C serum ,Dry skin: Use 
moisturizer and drink more water, Dandruff: Use anti-dandruff shampoo, Hair fall: 
Apply hair oil) 
• The system gives AI-based care suggestions to help improve the user’s skin and hair 
condition. It tells users what daily care routine, products, or natural remedies they can 
use.

3   Specific Requirements  
3.1 Functional Requirements (FR) 
FR1: User Authentication 
The system shall provide secure user registration and login using email/phone + password. 
Users shall be authenticated via JWT-based session handling (Django REST). 
FR-2: Database Storage 
The system shall store: 
• User profile information (name, gender, age, medical info). 

 
 8 
Software Requirements Specification for Projects 
• Authentication credentials (securely hashed). 
• Uploaded images (face, scalp) to secure media storage. 
• Diagnostic results and reports. 
FR-3: User Image Input 
The system must allow users to upload or capture high-quality images of their face or scalp 
through a web or mobile interface. 
FR-4: Image Validation 
Before analysis, the system must validate image clarity, lighting, and file size (≤ 10 MB). If the 
image is invalid, the system prompts the user to retake or reupload. It also confirms before 
uploading is it scalp or skin  
FR-5: AI-Based Detection Pipeline 
The system shall use the following AI modules: 
• Face & Scalp Detection: Mediapipe. 
• Segmentation: U-Net 
• Condition Classification: EfficientNet-B4+ YOLOv8 
• Severity score: XGBoost 
• Recommendation Engine: LLM 
These models shall be integrated through Django REST + React.js + PyTorch/TensorFlow. 
FR-6: AI-Based Skin Condition Detection 
The AI module must detect facial skin issues such the system shall detect: 
• Acne 
• Dark spots 
• Hyperpigmentation 
• Skin dryness 
using pre-trained CNN models. 
FR-7: AI-Based Scalp Condition Detection 
• The system must analyze scalp images to detect Dandruff 
• Dryness 
• Oiliness 
• Indicators of hair fall 
related problems. 
FR-8: Severity Classification 
• Detected skin and scalp conditions must be categorized by Mild 
• Moderate 
• Severe.

FR-9: Personalized Diagnosis and Care Suggestions 
Based on the AI results and user-provided information (age, gender, lifestyle), the system must 
generate custom skincare and haircare suggestions such as product use. 
FR-10: Medical-History-Aware Treatment Recommendation 
The system must provide personalized treatment recommendations that consider the user’s 
medical history to ensure safety and accuracy. For example, if a user has diabetes or allergies, 
pregnancy, the system must automatically filter out unsafe products or routines and suggest 
medically appropriate alternatives. 
FR-11: Result Visualization and Report Generation 
The system must produce textual report showing detected issues, severity, and recommended 
treatments. 
FR-12: Result Storage and History Tracking 
The system shall provide a dashboard where the user can: 
• View past diagnoses 
• Track progress over time 
• Compare condition improvement historically 
FR-13: User Profile Management 
Users shall be able to: 
• Update their personal profile 
• Edit medical information 
• View saved reports 
• Manage uploaded images 
FR-14: Feedback and Model Improvement 
Users should be able to rate the accuracy of the diagnosis or recommendations, helping refine AI 
models over time. 
FR-15: Data Security and Privacy 
All stored data including medical information and uploaded images must be encrypted and 
accessible only to authorized users. 
FR-16: Error Handling 
The system must display informative error messages when encountering poor quality images, 
network issues, or internal processing errors. 
FR-17: Safe Recommendation Enforcement 
The system shall ensure that all skincare and scalp care recommendations are medically safe and 
avoid suggesting harmful or risky products.

The system shall provide a simple explanation or reasoning (e.g., highlighted region, detected

The system shall provide educational tips related to lifestyle, environment, and habits that may

standard internet speed.

datasets.

• 
 Scalability: The Django-based backend must handle multiple concurrent users without 
degradation.

“View Report”).
 
• 
 Security: All user data (medical, images, reports) must be encrypted in transit and at rest 
using SSL/TLS and AES-256.
 
• 
 Availability: Maintain at least 99% uptime for 24/7 accessibility.
 
• 
 Maintainability: Codebase should be modular, documented, and version-controlled (Git) for 
easy model retraining and updates.

5.4 Communication Interfaces 
• The system uses the internet to send and receive data. 
• When a user uploads a photo, it is sent to the Django server through the REST API. 
• The Django server passes the photo to the AI model for checking. 
•  The AI model analyzes the photo and creates a report showing skin or scalp problems. 
•  The report is sent back to the Django server, which then stores it in the database and shows 
the result to the user.

6 Design Constraints  
• The system is built using the following technologies: 
o Frontend: React 
o Backend: Django to process images and manage data. 
o AI Tools: Google coolab, clickup, visual 
o Database: MySQL to save user photos, reports, and login data. 
• The system uses a client–server structure: 
Client side (user side): The user can take or upload a photo of their face or hair and see 
the results. 
Server side (Django): It receives the photo through the REST API, sends it to the AI 
model to check skin or scalp problems, then sends the report back to the user. 
•   Requires a camera and a stable internet connection to upload and analyze photos correctly.
